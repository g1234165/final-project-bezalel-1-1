# PRD — תיקון רספונסיביות דסקטופ | אתר פרויקט גמר, בצלאל

**גרסה:** 1.0 · **תאריך:** 06.07.2026 · **קבצים בסקופ:** `index.html`, `style.css`
**מחוץ לסקופ:** מובייל וטאבלט (אין תמיכה), `script.js` למעט התאמות Matter.js בפאנל 5.

---

## 1. עמודים (Pillars) — מטרת התיקון

**המטרה:** האתר ייראה מעולה, יציב וזהה-ברוחו לעיצוב הפיגמה בכל מסך דסקטופ נפוץ — מלפטופ 13" (1280–1366px) ועד מסך 27" (2560px) — בלי שאלמנטים זזים, עולים אחד על השני, או שלוגואים וצורות גרפיות מתעוותים.

1. **יציבות גיאומטרית** — אלמנטים גרפיים (לוגואים, כוכבים, צורות SVG, פופאפים) שומרים על פרופורציה וגודל קריא בכל רזולוציה. גרפיקה לא "נמתחת" עם המסך.
2. **גמישות מבוקרת** — רק אזורי התוכן (טקסט, גרידים, אזורי גלילה) סופגים את הפרשי הרוחב בין מסכים. המבנה (אקורדיון, טאבים של 80px, sidebar-ים) נשאר קשיח.
3. **אפס חפיפות** — אף אלמנט לא עולה על אלמנט אחר ואף טקסט לא נחתך בשקט. תוכן שלא נכנס — נגלל, לא נדחס.
4. **נאמנות לקונספט** — הרזולוציה הרפרנסית היא קנבס הפיגמה **2048×1070**. ברזולוציה הזו האתר חייב להיראות פיקסל-פרפקט כמו היום; בשאר הרזולוציות — אותה אווירה, אותם יחסים ויזואליים.

**הבעיה הנוכחית (Root Cause):** תרגום ישיר מפיגמה הותיר בקוד **152 שימושים ב-`cqw`/`cqh`** (77 `cqw`, 75 `cqh`) — כולל על לוגואים, צורות ופופאפים. יחידות container-query מותחות *הכל* פרופורציונלית לשני הצירים בנפרד, ולכן ביחסי מסך שונים מ-1.914 (יחס 2048/1070) הפרופורציות הפנימיות נשברות: גרפיקה מתעוותת, ריווחים אנכיים קורסים, ואלמנטים חופפים.

## 2. מגבלות (Constraints)

### 2.1 חובה — מעבר למידות קבועות באלמנטים גרפיים
- כל לוגו, צורת SVG, כוכב, אייקון ופופאפ עוברים ל-**`px` קבועים** (או `rem` כשמדובר בגודל שנגזר מטיפוגרפיה), עם `aspect-ratio` מפורש כשצריך.
- טבלת המרה מחייבת (לפי קנבס 2048×1070):
  - `1cqw = 20.48px` → לדוגמה: `--nav-w: 3.906cqw` → **`80px`**, `--sidebar-w: 22.949cqw` → **`470px`**
  - `1cqh = 10.7px` → לדוגמה: `--sp-pad: 4.484cqh` → **`48px`**, `--sp-content: 7.477cqh` → **`80px`**
- טיפוגרפיה נשארת ב-`clamp()`, אבל הערך האמצעי מוחלף מ-`cqw` ל-`vw` או לערך `px` לפי breakpoint (ראו 2.4). אסור להשאיר `cqw` בתוך `clamp` של פונטים.

### 2.2 חובה — `min-width` / `min-height` לאזורי תוכן
- לכל עמודת תוכן ראשית (sidebar של כל פאנל, אזור הרופאים בפאנל 1, אזורי הגלילה בפאנלים 3 ו-5) יוגדרו `min-width` ו/או `min-height` בערכי `px` שמתחתם התוכן לא נדחס אלא גולל.
- ערכי `min-height` הקיימים ביחידות `cqh` (למשל `min-height: 17.196cqh`, `min-height: 199.0cqh`) מומרים ל-`px` לפי הטבלה בסעיף 2.1.
- כללי flex ההגנתיים (`min-width: 0` / `min-height: 0` על ילדי flex גמישים) נשמרים כפי שהם — הם נכונים.

### 2.3 חובה — ניהול חריגות (Overflow)
- כל אזור תוכן פנימי בפאנל שעלול לחרוג בגובה מקבל **`overflow-y: auto`** (כיום יש רק 5 מופעים — לא מספיק). אסור `overflow: hidden` על אזור שמכיל טקסט רץ.
- עיצוב scrollbar אחיד (דק, בצבע `--color-light` על `--color-dark`) כדי שגלילה לא תשבור את האסתטיקה.
- ה-`overflow: hidden` על `html, body` ועל `.panel` נשמר — הוא חלק ממנגנון האקורדיון.

### 2.4 חובה — Media Query למסכים קטנים (עד 1366px)
- נקודת שבירה אחת לפחות: `@media (max-width: 1366px)` שמכוונת ללפטופים 13"–14" (1280×800, 1366×768).
- בתוכה: סט מוקטן של design tokens בלבד (ריווחים, רוחב sidebar, גדלי גרפיקה) — **לא** שכתוב layout. אותו מבנה, מידות מצומצמות.
- מותר להוסיף breakpoint שני ל-`max-width: 1512px` אם בדיקות ה-QA יראו צורך — אבל לא יותר משתי נקודות שבירה.

### 2.5 אסור — פגיעה בעיצוב ובקונספט
- **אסור** לשנות: פלטת הצבעים (`#101212` / `#F8F4EE`), הפונטים (Simona Pro / Narkiss Asaf), מבנה האקורדיון, כיווניות ה-RTL, האנימציות והטרנזישנים הקיימים, התנהגות ה-hover של הטאבים, וקווי ההפרדה בעובי 1px.
- **אסור** להוסיף frameworks, ספריות CSS, או להמיר ל-Grid/מבנה אחר. וניל בלבד, BEM נשמר.
- **אסור** שהתיקון יגרום לרגרסיה ברזולוציה הרפרנסית 2048×1070 — שם האתר חייב להיראות בדיוק כמו היום.
- Matter.js בפאנל 5 נשאר; מותר רק להוסיף לו טיפול ב-resize (סעיף Phase 6).

## 3. שלבים (Phases)

### Phase 0 — מיפוי ובייסליין
1. צילומי מסך של כל 6 הפאנלים (0–5) פתוחים, ברזולוציה 2048×1070 — זה ה-baseline הוויזואלי.
2. הפקת רשימה מלאה של כל 152 מופעי ה-`cqw`/`cqh` (grep), וסיווג כל מופע לאחת משלוש קטגוריות:
   **(א)** גרפיקה/לוגו → יומר ל-`px` קבוע · **(ב)** ריווח/מבנה → יומר ל-token ב-`px` עם וריאנט ב-breakpoint · **(ג)** טיפוגרפיה → `clamp` עם `vw`.

### Phase 1 — Design Tokens חדשים
1. שכתוב בלוק `:root`: כל ה-tokens עוברים ל-`px` לפי טבלת ההמרה (2.1). לדוגמה:
   ```css
   :root {
     --nav-w:      80px;    /* היה 3.906cqw */
     --sidebar-w:  470px;   /* היה 22.949cqw */
     --sp-pad:     48px;    /* היה 4.484cqh */
     --sp-content: 80px;    /* היה 7.477cqh */
     --fs-body:    clamp(13px, 0.9vw, 18px);
     --fs-h2:      clamp(26px, 2.2vw, 44px);
     --fs-h3:      clamp(17px, 1.4vw, 28px);
   }
   ```
2. הסרת `container-type: size` מ-`.accordion` (לא נחוץ יותר אחרי סילוק יחידות ה-container).

### Phase 2 — קיבוע גרפיקה ולוגואים
1. כל לוגו, צורה, כוכב ופופאפ: המרה ל-`px` + `aspect-ratio` + `flex-shrink: 0`. לדוגמה, הפופאפים בפאנל 4:
   ```css
   .p4__popup--small { width: 480px; min-height: 302px; }
   .p4__popup--big   { width: 680px; min-height: 423px; }
   .p4__popup--xl    { width: 850px; min-height: 713px; }
   ```
2. תמונות SVG מקבלות `object-fit: contain` ומידות מקובעות — אף פעם לא רוחב באחוזים ללא הגבלת `max-width` ב-`px`.

### Phase 3 — שריון ה-Layout
1. הוספת `min-width` ל-sidebar-ים של כל הפאנלים (למשל `.p1__sidebar { flex-basis: var(--sidebar-w); min-width: 380px; }`).
2. המרת כל `min-height` שנותר ב-`cqh` ל-`px`.
3. וידוא ש-`min-width: 0` / `min-height: 0` קיימים על כל ילד flex שאמור להתכווץ (אזורי טקסט וגרידים) — כדי שהכיווץ ילך לתוכן הגליל ולא לגרפיקה.

### Phase 4 — ניהול Overflow
1. הוספת `overflow-y: auto` לאזור התוכן הראשי של כל פאנל (למשל `.p1__doctors-main`, `#p3-scroll`, `.p5__main` — ולכל אזור מקביל בפאנלים 0, 2, 4).
2. עיצוב scrollbar אחיד:
   ```css
   .scrollable { scrollbar-width: thin; scrollbar-color: var(--color-light) transparent; }
   ```
3. בדיקה שאף `overflow: hidden` לא חוסם תוכן טקסטואלי שגדל ב-1366px.

### Phase 5 — Media Query למסכים קטנים
1. הוספת בלוק יחיד שמצמצם tokens בלבד:
   ```css
   @media (max-width: 1366px) {
     :root {
       --sidebar-w:  380px;
       --sp-pad:     32px;
       --sp-content: 48px;
     }
     /* צמצומי גרפיקה נקודתיים לפי ממצאי QA, למשל: */
     .p4__popup--xl { width: 720px; }
   }
   ```
2. אין לשנות בתוך ה-media query שום דבר מבני — רק ערכים.

### Phase 6 — Matter.js (פאנל 5)
1. הסימולציה קוראת את מידות ה-`.p5__sidebar` פעם אחת ב-`startPhysics()`. אחרי המעבר ל-`px` המידות יציבות יותר, אבל חובה להוסיף האזנה ל-`resize` עם debounce (250ms) שמריצה `stopPhysics()` → `startPhysics()` כשהפאנל פעיל, כדי שהקירות והרצפה יתאימו לגודל החדש.
2. הגדרות ה-`wF` וה-`aspect` של הצורות נשארות כפי שהן — הן כבר יחסיות ל-frame.

### Phase 7 — QA ו-Definition of Done
בדיקה ידנית של כל 6 הפאנלים בכל אחת מהרזולוציות במטריצה שבסעיף 4.

### Definition of Done — האתר "גמור" כאשר:
1. ✅ `grep -c 'cqw\|cqh' index.html` מחזיר **0** (וכך גם ב-`style.css`).
2. ✅ בכל רזולוציה במטריצה: אפס חפיפות אלמנטים, אפס טקסט חתוך, אפס גרפיקה מעוותת (בדיקה ויזואלית פאנל-פאנל).
3. ✅ ברזולוציה 2048×1070 — התאמה מלאה לצילומי ה-baseline מ-Phase 0.
4. ✅ לוגואים וצורות SVG שומרים על יחס גובה-רוחב מדויק בכל רזולוציה (מדידה ב-DevTools).
5. ✅ בפאנל 5: הצורות נופלות, נערמות ונגררות תקין גם אחרי שינוי גודל חלון תוך כדי צפייה.
6. ✅ אנימציית האקורדיון (פתיחה/סגירה של פאנלים) חלקה וללא קפיצות layout בכל הרזולוציות.
7. ✅ אין רגרסיה בהתנהגויות קיימות: hover על טאבים, ציטוטי הרופאים בפאנל 1, האקורדיונים הפנימיים בפאנלים 3 ו-5.
